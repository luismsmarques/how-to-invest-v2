# content-editorial ÔÇö Glossary, news review & the content pipeline

Covers the editorial surface **around** the Learn hub. For Learn chapters themselves use `learn-guide`; for search structure use `seo-wordpress`. Anchor doc: `docs/Estrategia_Conteudo_SEO_LLM.md` (hub-and-spoke: Learn pillars + glossary spokes + news).

## Non-negotiable invariants
- **Asset-class only** (no named instruments/tickers/funds/companies), **educational/conditional tone**, **disclaimer** present, **bilingual EN + PT**. News drafts are **never auto-published** ÔÇö human review is the real gate.

## Two-stage content model
- **Seeder = install-only** (`includes/class-seeder.php`): starter glossary + institutional/legal pages (301 targets). Idempotent, create-only ÔÇö it never updates existing posts. Don't use it for ongoing edits.
- **Importers = ongoing editorial** (idempotent by slug, Polylang-linked, publish immediately; a re-sync never reverts an editor's status change).

## Glossary authoring
- One Markdown file per term: `wp-content/plugins/hti-engine/content/glossary/<slug>.md`.
- Frontmatter: `slug`, `slug_pt`, `topic`, `term_en`, `term_pt`, `excerpt_en/pt`, `seo_title_en/pt`, `seo_desc_en/pt`. Body split `<!-- EN -->` / `<!-- PT -->`: a leading `> ` TL;DR, `## ` question headings, a `## Key takeaways` / `## Pontos-chave` list.
- **Internal-link tokens** `[glossary:slug|Text]` / `[learn:slug|Text]` resolve via `class-content-import.php::inline()`. **Convention: put tokens only in the EN section** (PT stays plain prose). Every referenced slug must exist ÔÇö **no dangling tokens** (`grep -rhoE "\[learn:[a-z0-9-]+\|" content/glossary/`).
- Cluster rule: each glossary term links to its **pillar** Learn chapter; each pillar links to its spoke terms (`glossary` frontmatter drives the localized "Related terms" rail).
- Import: `wp hti import-glossary` (or Tools Ôćĺ Glossary content).

## News (hti-rss-ai) editorial
- News is machine-generated by `wp-content/plugins/hti-rss-ai/` (see its `README.md`; feeds the `news` CPT). Drafts land **pending review** ÔÇö review, edit and publish via `class-drafts.php` / `class-review.php`.
- `class-validator.php` rejects advice/tickers/wrong-language/near-copies and clamps the meta description ÔÇö but it is a safety net, **not** the editorial gate. Publish **few and good**, not many thin.
- Per-item generation (News/Quote/Tutorial/Summary) exists on any draft (`class-generator.php::generate_from_item`).

## Pipeline reference
- `class-content-import.php` (Learn) + `class-glossary-import.php` (reuses `to_blocks`) ÔÇö parse `.md` Ôćĺ upsert EN+PT, taxonomy, TL;DR + takeaways, quiz meta.
- Spine: `content/learn-plan.csv` (`module,order,slug,ÔÇŽ,prev,next`); nav (prev/next) is derived from module/order order, not the frontmatter fields.
- Commands: `wp hti import-learn | import-glossary | seed` (or the Tools screen).

## Checklist (before "done")
- [ ] Asset-class only; conditional tone; disclaimer; **EN + PT**
- [ ] Glossary tokens in the EN section only; **no dangling `[learn:]`/`[glossary:]`**
- [ ] Term linked to its pillar; pillar links back (cluster complete)
- [ ] News reviewed by a human before publish; few-and-good, not thin
- [ ] Imported via the pipeline (not hand-edited in the DB); re-import is idempotent
